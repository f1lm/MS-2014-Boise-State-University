History

Scala began in 2001 as an attempt to fix many of the so-called “poor choices” that Java made on the front end. It was conceived and initially implemented by lab researchers at the Swiss Federal Institute of Technology in Lausanne. The head of the project, Martin Odersky, had worked previously on a project called Generic Java, which acted as a superset of Java with additional generic programming abilities. He had also worked on javac, Sun’s original Java compiler, and still the most popular for the language. It is easy to see why he might be interested in improving the language he’d spent so long working around.

One big emphasis in the Scala project was creating a unified type system, where all types inherit from a single root object. Java allows both primitives and Objects, which are treated separately. This will be discussed in more detail in the Data Types section below. Scala is compiled to Java bytecode on the backend, which is commonly regarded as quite good. This means libraries between Java and Scala are interchangeable.

Another emphasis was to add many features from functional programming languages, such as type inference, list comprehensions, anonymous functions, and many more. These were borrowed from languages such as Scheme and Haskell.

Scala was released in late 2003 using the JVM, and later in 2004 on the .NET platform. It remained relatively unknown, however, until it won a large research grant of 2.3 million Euros in 2011 (about 3.4 million dollars at the exchange rate that year) from the European Research Council. Since then, the commercial use of the language has increased fivefold, and it appears to be gaining more traction as a Java alternative. It is helpful that Scala also runs on Android, enabling it to be a factor in the upcoming mobile wars.

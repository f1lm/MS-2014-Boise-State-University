Type Parametrization

  • Type parametrization is like generics in Java; it allows programmers to create types that takes another type as parameter. 

    		  
		    def position [A] {lis: List[A], value :A} : Int ={ 
			lis.indexOf {value}  
		    }

    In this example, the function position return index of value in List lis. The value of type A will be determined when the function is invoked

  • In Scala, parametrization is complemented by Type variance with specified constraints like covariance and contravariant.

  • Covariance allows you to go from generic or broader type to a narrower the type in covariant position (i.e return type). A popular application of covariance is an immutable list; an immutable list is covariant in its type parameter. For example, List[Int] is a subtype of List[Any]. Covariance is useful when you return a collection. For example, in Scala, Traversable is the parent trait for all the collection types. Because Traversable is defined with a covariant parameter, you will get the desired type of collection back even you use Seg, Iterable or List (which inherit the definition from Traversable). You denote covaraiance with + sign. 

    		  
		    trait Foo[+R] {}

    In this example, foo is the covariant in R.

  • Contravariance allows programmer to go from a narrower type to a more generic type. You can denote contravariance by using the - sign:

    		  
		    trait Foo[-P,+R] {}

    In the example, foo is contravariant in P and covariant in R. We can implement foo in the following manner:

    		  
		  val add: Foo[Any, Int]= {x: Int=> x+10}

    Foo can take object as a parameter because Any is the superclass of all classes. However, it is invoked only if x is an Int. Remember that we cannot use a contravariant return type

Interpreter

  • Scala has an Interpreter. It is an interactive shell called scala or REPL( stands for Read-Evaluate-Print Loop). The lastest version is included in the scala 2.11.0-m5 package (release date: September, 2013). It is improved with several new commands that ease its usage. In this introduction, we still use version 2.10.0 of the interpreter. You can use it by typing scala, at a command prompt: 

    •               
               $scala

    You will receive the following message :

                  
               Welcome to Scala version 2.10.0.final (Java ...).
	       Type in expressions to have them evaluated.
	       Type :help for more information.
	       scala>

    This means the Scala installation is successful and now you can run your program

  • The table bellows shows important Scala interpreter commands:

    Command Description
    :help This command prints the help message with all the commands available in the Scala interpreter
    :cp Use this command to add a JAR file to the classpath for the Scala interpreter.
    :load or :l Scala code, you could load the file into the Scala interpreter, and all the definitions will be accessible to you.
    :replay or :r Resets the interpreter and replays all the previous commands.
    :quit or :q Exits the interpreter.
    :type Displays the type of an expression without evaluating it.

  • Scala can run on the Java Virtual Machine because Scala can be compiled to Java byte code. Also, the Scala compiler contains a subset of the Java compiler and their classes can be freely mixed.

Casting

  • When you want to cast a variable of one primitive type to another primitive type, you can use .toInt, .toDouble, ect. For example, if you want to cast a float to Int

                  
				val aFloat = 1.5f
				val i : Int = aFloat.toInt

  • You can use explicit casting if you want to cast object or variable of any type (including the primitive types). For example, you want to cast an Object type to a String type 

    		  
		    val a java.lang.Object = new java.lang.String("hello World")
		    val val s: String = a.asInstanceOf[String]

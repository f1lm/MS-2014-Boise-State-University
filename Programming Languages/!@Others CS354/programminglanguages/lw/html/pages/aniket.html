Scala Program Compilation

Scala provides a few ways to run its program files. Here we'll look at manual compilation of .scala files and gain some insight in how Scala complies its files.

  • Instead of using the Scala REPL "Interpreter" you can manually compile .scala files and then run them just like in Java like this: 

    			
              $ scalac test.scala
              $ scala test

    $ is a command line symbol in your terminal. You don't type this. The scala command will run your scala program after it is compiled.

    You can compile multiple files like this: 

    			
              $ scalac <file1> <file2> ...

    or like this: 

    			
	      $ scalac *.scala

    *.scala means all .scala files in the directory. Then use the scala command to run your main .class file.

    We could run a .scala file without compiling it first like this: 

    			
	      $ scala test.scala

    Then Scala would compile and run the file. But constantly doing this to run a file is slow. By manually compiling .scala files first, Scala won't need to compile them every time we run them. Therefore, it is faster. And manual complication is a lot faster than loading the file into the Scala REPL "Interpreter" using :load and running it.

  • How you write a .scala file also determines how you should run it.

    When you write a .scala file, you can write it as a script or a non-script. A script is a file with a sequence of statements that ends in a result expression. A script can contain class definitions, expressions, etc. as long as it ends in a result expression.

    For example, let's assume the following statement is in a file called hello.scala : 

    				
	      println("Hello!")

    You can then run the script like this:

    			   
	     $ scala hello.scala

    This will print:

    			   
	     Hello!

    Script files can also be run using the REPL Interpreter.

    If you write a non-script .scala file that only contains class definitions and no result expressions. You will have to run your program by compiling all the necessary files using scalac command. Then running the file that contains the main program that uses those classes with the scala command. The main program file must contain a singleton object with a main method.

    For example if we had a file progRunner.scala that runs the program using the classes we defined. The file would look like this:

    			   
	object progRunner{
	  // optional function definitions that uses classes.
			
	  def main(args: Array[String]) : Unit = {
	   //do stuff with classes or run functions defined above
	  }
	}

    After compiling all files including progRunner.scala, we then type the following to run our program or application: 

    			   
	     $ scala progRunner

    The def main(args: Array[String]) : Unit function is similar to Java's public static void main(String [] args) method.

  • Scala compilation works by transforming Scala code into Java bytecode to run on the Java Virtual Machine (JVM)

    Even when you use the Scala REPL (Read Evaluate Print Loop) "Interpreter", Scala is taking the source code you enter and immediately compiling it, then running it, and then printing the result. So the Scala "Interpreter" is very different from an actual Interpreter used in other functional languages like Scheme. Scala is a compiled language not an interpreted language so it always uses a compiler, hence the " " around Interpreter. However, Scala is good at imitating an interpreter with the REPL.

Control Flow

The Control Flow in Scala is similar to the one in Java but a little different.

  • There is a linear ordering of statements in Scala. That is, one statement follows another. Statements can be organized with {} just like in Java.

    For example:

    			   
	  

var x = 3;

if(x > 2){ 
	var y = x + 4;
	x + y; 
}else{ 
	var z = 2;
	x + z;
}

    First var x = 3; is evaluated then the if statement.

    Inside the braces of the if statement the statements are evaluated in order.

  • If and If-Else Statements

    If statements in Scala are written like this if(cond) stmt or if(cond) {stmts...}

    If statements execute the following statement or statements if the condition (cond) is met or evaluates to true. The "cond" between the "()" is where a boolean expression would go. A boolean expression is an expression that evaluates to true or false.

    Here are some basic boolean expressions that are used with numbers:

    			   
1 > 2 // greater than. returns true if number on left is greater than number on right.
1 >= 2 // greater than or equal to.
1 <  2 // less than. 
1 <= 2 // less than or equal to.
1 == 2 //equals. returns true if number on left is equal to number on right.
1 != 2 // not equal

    Let's use these boolean expressions in an if statement:

    			   
if(1>2) {println("Hello")}

    The if statement above would not evaluate the statement between the braces because 1 is not greater than 2.

    			   
if(1<2) println("Hello")

    Now the condition is true so Scala will print out Hello.

    If-Else Statements are just like if statements but allow an alternate option if the condition is false. They are written like this if(cond) stmt else stmt or if(cond) {stmts...} else {stmts...} If "cond" returns true then the statment(s) after the if are evaluated. If "cond" returns false then the statement(s) after the else are evaluated.

    For example:

    			   
if(1<2) 3 else 4

    The if-else statement above will print 3 since 1 is less than 2

    			   
if(1>2) 3 else 4

    The if-else statement above will print 4 since 1 is not greater than 2 so the cond returns false.

    Scala also supports && and || operators to be used with boolean expressions.

    && is the And operator.

    			   
<boolexp1> && <boolexp2>

    The statement above returns true if both boolexp1 and boolexp2 return true otherwise it returns false.

    || is the or operator

    			   
<boolexp1> || <boolexp2>

    The statement above returns true if either boolexp1 or boolexp2 returns true. If both boolexp1 and boolexp2 return false, then the statement returns false. If boolexp1 and boolexp2 both return true, then the statement also returns true.

  • While Loop

    while (cond) <block/exp> allows you to repeat a block of code or an expression while the condition is true:

    			   
var x = 10
while (x < 20) x = x + 3

    In the above statement, x = x + 3 will continue to evaluate until x < 20 is false. If x < 20 never becomes false then you have an infinite loop because x = x + 3 will continue to run forever. You have to stop the program manually or restart your computer.

  • Do While Loop

    do <block/exp> while (cond) allows you to repeat a block of code or an expression while the condition is true. The condition is evaluated after doing each iteration:

    			   
var x = 10
do {
x = x + 3
} while (x < 20)

    Do While Loops guarantee the statement inside the braces or after "do" to be executed at least once.

  • For Loop

    for (range) <block/exp> allows you to repeat a block of code for all the values in a range or iterate through the members of a collection.

    			   
for(i <- 1 to 5) println("hi")

    The statement above takes values 1 through 5 and puts them in a variable i one by one after each repetition of the println statement. hi will be printed 5 times.

    If you want the end range value not to be included the until version should be used.

    			   
for(i <- 1 until 5) println(i)

/* 
Scala will print:
1
2
3
4
 
*/

    Multi-dimensional iterations are elegantly handled using multiple ranges. Notice that the two ranges are separated by a semi-colon.

    			   
for(i <- 1 until 4 ; j <- 1 to 3) println(i,j)

/* 
Scala will print:
(1,1)
(1,2)
(1,3)
(2,1)
(2,2)
(2,3)
(3,1)
(3,2)
(3,3)

*/

    For Loops may also be used to iterate through collections like a List:

    			   
var x = List(1, 2, 3)

for( i <- x) println(i)

/* 
Scala will print:
1
2
3

*/

  • Match

    Match statement in Scala is similar to Java's switch statement. It uses 'case' statements to allow multi-way branching on value.

    			   
def decode(n: Int) {
	n match {
		case 1 => println("One")
		case 2 => println("Two")
		case _ => println("Error")
	}
}

decode(2)

/* 
Scala will print:
Two

*/

    In the above example we created a function that takes in an Int argument and puts it in variable n. The variable n is then used in the match statement to determine which case to perform.

    The '=>' symbol is used to separate the match pattern from the expression or block to be evaluated. The '_' symbol is used in Scala to mean wild-card or in this case match anything. The last case statement behaves like default in the classical switch.

    Unlike the traditional Java 'switch' the above mapping can easily be reversed.

    			   
def encode(s: String){
	s match {
		case "One" => println(1)
		case "Two" => println(2)
		case _ => println(0)
	}
}

encode("Two")

/* 
Scala will print:
2

*/

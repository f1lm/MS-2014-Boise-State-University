Cookbook

How to operate

General

As a server application, the cookbook is depending on the capability of your browser. A resemblance with Windows-Applications is wanted, however limited. Particularly, the processing of many windows is put in the background. Instead of it, a nesting of the application-functions with a return-logic is important. This handling is under experts of mainframes very familiar.

Remark: within the application, the use of "back" function (like the "back" button of your browser) is not supported. In this case, please use the "back" button of the application. So you go back to the page you came from.

Toolbar

The toolbar helps for a quick navigation to the most frequented functions. It finishes the preceding step and starts the desired function. In this case, a return to the preceeding function is not possible.
 
 
[icon menu]  Branches to the main menu
[icon enquiry]  Starts the cookbook inquiry
[icon close]  Finishes the application and closes the server connection. This is a preferred way to close the application, instead to close the window.

Menu


 

The menu is hierarchical builded and comprises many steps. With the button "Back", you go just one step back. The branch is based on cursor and click.
 

Enquiry/Search within the cookbook

Helps you to find recipes based on to one or more search arguments. Put your limitations and press button "Listing".

Remarks:

  • The combination of limitations is allowed. It helps to restrict the search and get exact results.
  • All given criterions are "AND-conditions", this means, the more you give criterions, the less results you get.
  • All search criterions are interpreted as generic, e.g. if you enter "carrot" in the field "recipe-type", you will find "carrot salad" and "carrot soup", if they exist. If you enter "carrot" in the field "ingredients", a list of all recipes that contains the word "carrot" is displayed.

Enquiry-results

All found recipes (or limited until a defined number) are displayed and are ready for further-processing. When you click on one of these recipes, a detailed display of this recipe is shown.


 
 

Login for more functionality

Without a login into the application, only basic functions are allowed and shown. For more functionality, you have to log in. This sample shows possible enlargement of authorizations:
 
Standard: Read/Modify/New/delete of Recipes
Org: Read/Modify/New/delete of Recipe-Types and Recipes. 
Admin: Read/Modify/New/delete of Users/Groups, Recipes and Recipe-Types

New Recipe

Befor you enter a new recipe, be sure that this does not exist. This avoids duplicate. Enter your data and press "OK" to insert this recipe. The button "cancel" cancels your data and no recipe is created.


 
 
 
Recipe-Title The title of your recipe e.g. "vegetable soup"
Recipe-Type The type of your recipe e.g. "Soup"
Diabetic-
recommendation		 Mark this field with a cross if this recipe is diabetic recommended.
Ingredients Ingredients of the recipe
Preparation The way how you prepare this recipe
Source Enter the source from where is your recipe.
Origin Date The date of the first publication

Change of recipe

It also is possible to change an already existing recipe. After searching and finding it, press the button "process" to be able to change. When done, press the button "OK" to save your changes. The button "cancel" cancels your changes.


 
 
 

Deletion of recipes

If required, you also may delete a recipe. After searching and finding it, press the button "delete" to delete it. The button "cancel" cancels this action.


 
 
 

New type of recipes

At this place, you can enter your new type of recipe in the field "description". To sort your types by appearance in the select boxes, type a value in the field "key".

Types are e.g. "soups", "salads" or "desserts".


 
 
Description enter the name of the recipe type. This is a mandatory field
Pos(Position) To sort your types by appearance in the select boxes, type a value. If you keep the field blank, the display is in an alphabetic order.

Confirm your data with the Button "OK".

New user and user group

The menus "Administration" and "User administration" allows you to create userIds and user-groups. Each userid belongs to a user-group with specific auhorization.

Confirm your data with the button "OK".
